## Шпаргалка — Как считать VLSM

> [!important] Алгоритм VLSM — 5 шагов
> 
> 1. Отсортируй подсети от **большей к меньшей** по количеству хостов
> 2. Для каждой подсети найди минимальное **2ⁿ ≥ (нужно хостов + 2)**
> 3. Определи **prefix** = 32 − n
> 4. Network address = следующий свободный адрес после предыдущей подсети
> 5. Broadcast = network address + (2ⁿ − 1)

---

### Формулы

|Что считаем|Формула|
|---|---|
|Минимальный размер блока|2ⁿ ≥ (hosts + 2)|
|Usable hosts|2ⁿ − 2|
|Prefix length|/( 32 − n )|
|Broadcast address|Network + 2ⁿ − 1|
|First usable|Network + 1|
|Last usable|Broadcast − 1|
|Следующая подсеть начинается с|Broadcast + 1|

---

### Таблица степеней двойки

|n|2ⁿ|Usable|Prefix|Subnet Mask|
|---|---|---|---|---|
|1|2|0|/31|255.255.255.254|
|2|4|2|/30|255.255.255.252|
|3|8|6|/29|255.255.255.248|
|4|16|14|/28|255.255.255.240|
|5|32|30|/27|255.255.255.224|
|6|64|62|/26|255.255.255.192|
|7|128|126|/25|255.255.255.128|
|8|256|254|/24|255.255.255.0|
|9|512|510|/23|255.255.254.0|
|10|1024|1022|/22|255.255.252.0|
|11|2048|2046|/21|255.255.248.0|
|12|4096|4094|/20|255.255.240.0|

> [!tip] Быстрый способ найти prefix Считай сколько бит нужно для хостов (n), потом отнимай от 32. Нужно 600 хостов → 2¹⁰ = 1024 → n=10 → prefix = 32−10 = **/22**

---

### Пример расчёта шаг за шагом

> [!note] Пример: нужно 100 хостов, базовая сеть 10.0.0.0/8

```
Шаг 1: 100 + 2 = 102 (добавляем network + broadcast)
Шаг 2: 2⁷ = 128 ≥ 102  →  n = 7
Шаг 3: prefix = 32 − 7 = /25
Шаг 4: маска = 255.255.255.128
Шаг 5: если предыдущая подсеть закончилась на 10.0.3.255
        → эта начинается с 10.0.4.0
Шаг 6: broadcast = 10.0.4.0 + 128 − 1 = 10.0.4.127
Шаг 7: usable = 10.0.4.1 – 10.0.4.126
```

---

### Как определить маску из prefix (и обратно)

```
/24  →  последний октет: 256 − 2⁸ = 256 − 256 = 0    →  255.255.255.0
/25  →  последний октет: 256 − 2⁷ = 256 − 128 = 128  →  255.255.255.128
/26  →  последний октет: 256 − 2⁶ = 256 − 64  = 192  →  255.255.255.192
/27  →  последний октет: 256 − 2⁵ = 256 − 32  = 224  →  255.255.255.224
/28  →  последний октет: 256 − 2⁴ = 256 − 16  = 240  →  255.255.255.240
/30  →  последний октет: 256 − 2²  = 256 − 4   = 252  →  255.255.255.252
```

> [!tip] Формула для последнего октета маски **256 − 2ⁿ** где n = количество бит под хосты

## Термины

**VLSM (Variable Length Subnet Mask)** — метод подсетевой адресации при котором разные подсети могут иметь маски разной длины. Позволяет экономно расходовать адресное пространство. #abbreviation

**subnet** — подсеть, логически выделенный сегмент IP сети. #networkterm

**prefix length** — длина префикса сети, например `/24`. Показывает сколько бит отведено под сеть. #networkterm

**usable hosts** — количество используемых адресов в подсети = 2ⁿ − 2 (минус network address и broadcast). #networkterm

**broadcast address** — последний адрес подсети. Пакет на этот адрес получают все устройства в подсети. #networkterm

**network address** — первый адрес подсети. Идентифицирует подсеть, не назначается устройствам. #networkterm

**static route** — маршрут добавленный вручную администратором. Роутер не узнаёт о нём автоматически. #networkterm

**default gateway** — IP адрес роутера, на который устройство отправляет трафик для других сетей. #networkterm

**SVI (Switch Virtual Interface)** — виртуальный интерфейс коммутатора для управления по сети. #abbreviation

**point-to-point link** — линк между двумя роутерами. Использует /30 маску — только 2 usable адреса. #networkterm

## Step 2 — Таблица IP адресации

> [!important] Правила назначения адресов
> 
> - **Rule 4:** R1 g0/0/0 = 1st usable Network A; R2 g0/0/0 = last usable Network A
> - **Rule 5:** Роутеры получают **1st usable** своей подсети
> - **Rule 6:** Коммутаторы получают **2nd usable** своей подсети
> - **Rule 7:** Конечные устройства получают **последние usable** адреса
> - **Rule 8:** PC-A получает **3rd usable** Network B = 10.1.4.131

## Step 3 — Настройка роутеров

### R1

```
enable
configure terminal

interface g0/0/0
 ip address 10.1.4.161 255.255.255.252
 no shutdown
exit

interface g0/1
 ip address 10.1.4.129 255.255.255.224
 no shutdown
exit

end
write memory
```

### R2

```
enable
configure terminal

interface g0/0/0
 ip address 10.1.4.162 255.255.255.252
 no shutdown
exit

interface g0/1
 ip address 10.1.0.1 255.255.252.0
 no shutdown
exit

interface g0/2
 ip address 10.1.4.1 255.255.255.128
 no shutdown
exit

end
write memory
```

`ip address 10.1.4.161 255.255.255.252` #ciscoIOScommand Назначает IPv4 адрес и маску на интерфейс роутера. Маска `/30` = `255.255.255.252`.

`no shutdown` #ciscoIOScommand Включает интерфейс. По умолчанию интерфейсы роутера выключены (administratively down).

> [!warning] Маски — не перепутай!
> 
> |Prefix|Subnet Mask|
> |---|---|
> |/22|255.255.252.0|
> |/25|255.255.255.128|
> |/27|255.255.255.224|
> |/30|255.255.255.252|

## Step 4 — Настройка коммутаторов (SVI)

### S1 — Network B (/27)

```
enable
configure terminal

interface vlan 1
 ip address 10.1.4.130 255.255.255.224
 no shutdown

end

```

```
enable

end

```

### S3 — Network C (/22)

```
enable
configure terminal
```

## Step 6 — Статические маршруты

> [!important] Почему S1/S2/S3 не пингуют друг друга без статических маршрутов? Роутеры знают только о **напрямую подключённых** сетях. R1 не знает о сетях C и D (они за R2), R2 не знает о сети B (она за R1). Нужно добавить статические маршруты вручную.

### На R1 — добавить маршруты к сетям за R2:

```
enable
configure terminal

ip route 10.1.0.0 255.255.252.0 10.1.4.162
ip route 10.1.4.0 255.255.255.128 10.1.4.162

end
write memory
```

### На R2 — добавить маршрут к сети за R1:

```
enable
configure terminal

ip route 10.1.4.128 255.255.255.224 10.1.4.161

end
write memory
```

`ip route 10.1.0.0 255.255.252.0 10.1.4.162` #ciscoIOScommand Статический маршрут: пакеты для сети 10.1.0.0/22 отправлять на next-hop 10.1.4.162 (R2). Формат: `ip route [сеть] [маска] [next-hop]`.

> [!danger] Exam trap — next-hop адрес Next-hop должен быть IP адрес **соседнего роутера** на линке между ними, а не адрес назначения. R1 смотрит на R2 через 10.1.4.162, R2 смотрит на R1 через 10.1.4.161.
