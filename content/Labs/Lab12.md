> [!abstract] Суть лабораторной Настроить статическую маршрутизацию между 4 роутерами (R1-R4), обеспечить связь между сетями F103, F107, Canteen и Internet, а также настроить резервные маршруты через R4.

## Топология

```
PC-F103 → S1 → R1 → R2 → R3 → S3 → FTP
                          ↑         (Canteen)
Laptop-F107 → S2 → R1    R3 → Internet
                   ↕
                   R4
```

## Part 3 — Static Route к Network Canteen

> [!important] Логика маршрута Пакет идёт: `PC-F103 → R1 → R2 → R3 → FTP` Каждый роутер должен знать **следующий шаг (next-hop)** к сети назначения `172.16.1.0/24`

**На R1** (next-hop = R2 = `10.1.12.2`):

```cisco
conf t
ip route 172.16.1.0 255.255.255.0 10.1.12.2
end
```

**На R2** (next-hop = R3 = `10.1.23.3`):

```cisco
conf t
ip route 172.16.1.0 255.255.255.0 10.1.23.3
end
```

**На R3** (Canteen напрямую подключён через `g0/0`):

```cisco
conf t
ip route 172.16.1.0 255.255.255.0 g0/0
end
```

> [!note] Почему trace ещё не работает? R3 не знает обратный путь к сетям 192.168.1.0 и 192.168.2.0. Это исправим в Part 4.

**Результат: 50%** ✅

## Part 4 — Supernet + Summary Static Routes

### Шаг 1: Расчёт суперсети

Объединяем две сети:

```
192.168.1.0/24
192.168.2.0/24
```

**Переводим в двоичный вид:**

```
192.168.1.0 = 11000000.10101000.00000001.00000000
192.168.2.0 = 11000000.10101000.00000010.00000000
```

**Находим общую часть:**

```
11000000.10101000.000000|01.00000000
11000000.10101000.000000|10.00000000
                        ↑
                  здесь отличаются → 22-й бит
```

**Маска /22 в десятичном:**

```
11111111.11111111.11111100.00000000 = 255.255.252.0
```

> [!tip] Быстрый способ найти суперсеть Смотри где **первый отличающийся бит** — это и есть граница маски. 1 → 00000001, 2 → 00000010 → отличаются в 7-м бите третьего октета → считай с начала: 22 общих бита → /22

**Итог:**

```
Суперсеть = 192.168.0.0/22  (255.255.252.0)
Покрывает: 192.168.0.0 — 192.168.3.255 ✅
```

### Шаг 2: Настройка маршрутов

Обратный путь: `FTP → R3 → R2 → R1 → PC-F103/Laptop-F107`

**На R3** (next-hop = R2 = `10.1.23.2`):

```cisco
conf t
ip route 192.168.0.0 255.255.252.0 10.1.23.2
end
```

**На R2** (next-hop = R1 = `10.1.12.1`):

```cisco
conf t
ip route 192.168.0.0 255.255.252.0 10.1.12.1
end
```

> 
> - PC-F103 ↔ FTP ✅
> - Laptop-F107 ↔ FTP ✅

**Результат: 64%** ✅

## Part 5 — Default Static Route на R3

**На R3:**

```cisco
conf t
ip route 0.0.0.0 0.0.0.0 s0/0/0
end
```

> [!note] Что значит `0.0.0.0 0.0.0.0`? Это маршрут "для всего остального" — если роутер не знает куда отправить пакет, он идёт через этот маршрут. Используется для выхода в Интернет.

> [!warning] netacad.com не открывается — это нормально По условию задачи у сервера не настроен IP/DNS. Просто двигайся дальше.

## Part 6 — Backup Route через R4

> [!important] Логика резервного маршрута Основной путь (AD=1) используется всегда. Резервный (AD=2) активируется **автоматически** только если основной упал.
> 
> ```
> Основной:  R1 → R2 → R3 → Canteen   (AD=1)
> Резервный: R1 → R4 → R3 → Canteen   (AD=2)
> 
> Основной:  R3 → R2 → R1 → F103/F107 (AD=1)
> Резервный: R3 → R4 → R1 → F103/F107 (AD=2)
> ```

### Таблица адресов для резервного пути:

|Откуда → Куда|Next-hop IP|
|---|---|
|R1 → R4|`10.1.14.4`|
|R4 → R3|`10.1.34.3`|
|R3 → R4|`10.1.34.4`|
|R4 → R1|`10.1.14.1`|

### Команды:

**На R1** — резервный маршрут к Canteen (AD=2):

```cisco
conf t
ip route 172.16.1.0 255.255.255.0 10.1.14.4 2
end
```

**На R4** — маршрут к Canteen через R3:

```cisco
conf t
ip route 172.16.1.0 255.255.255.0 10.1.34.3
end
```

**На R3** — резервный summary route через R4 (AD=2):

```cisco
conf t
ip route 192.168.0.0 255.255.252.0 10.1.34.4 2
end
```

**На R4** — маршрут к суперсети через R1:

```cisco
conf t
ip route 192.168.0.0 255.255.252.0 10.1.14.1
end
```

### Проверка:

```cisco
show running-config | include ip route
```

**R1 должен показать:**

```
ip route 172.16.1.0 255.255.255.0 10.1.12.2      ← основной AD=1
ip route 172.16.1.0 255.255.255.0 10.1.14.4 2    ← резервный AD=2
```

**R4 должен показать:**

```
ip route 172.16.1.0 255.255.255.0 10.1.34.3
ip route 192.168.0.0 255.255.252.0 10.1.14.1
```

**R3 должен показать:**

```
ip route 192.168.0.0 255.255.252.0 10.1.23.2     ← основной AD=1
ip route 192.168.0.0 255.255.252.0 10.1.34.4 2   ← резервный AD=2
ip route 0.0.0.0 0.0.0.0 Serial0/0/0             ← default route
```

> [!danger] Резервный маршрут не виден в `show ip route static` — это нормально! Команда показывает только **активные** маршруты. Резервный (AD=2) спит пока жив основной (AD=1). Используй `show running-config | include ip route` чтобы увидеть **все** маршруты включая резервные.
