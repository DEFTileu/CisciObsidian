# 2.3 Module Practice and Quiz
## 2.3.1 What did I learn in this module?
Если собрать модуль в одну картину, получается так:
### Frame Forwarding
Switch:
- смотрит на `source MAC` и учит таблицу;
- смотрит на `destination MAC` и выбирает `egress port`;
- flood'ит `unknown unicast`, `broadcast` и часто `multicast` traffic.
### Switching Methods
Ты должен различать:
- `store-and-forward` = full frame + error check;
- `cut-through` = low latency + possible bad frame forwarding.
### Switching Domains
Ты должен понимать:
- `collision domain` связан с shared medium и `half-duplex`;
- `broadcast domain` объединяет устройства, получающие Layer 2 broadcast;
- `router` делит broadcast domains.
> [!abstract] Самое важное для запоминания
> - `Ingress = frame enters`
> - `Egress = frame exits`
> - `MAC table = MAC -> port`
> - `Unknown unicast = flooding`
> - `Store-and-forward = checks FCS/CRC`
> - `Cut-through = lower latency`
> - `Router separates broadcast domains`
## 2.3.2 Quiz mindset
На quiz по этому модулю лучше идти с таким пониманием:
- switch forwards by `destination MAC`;
- switch learns by `source MAC`;
- `full-duplex` и `half-duplex` влияют на collision behavior;
- interconnected switches expand one Layer 2 broadcast domain until a router breaks it.
> [!success] Главная идея Module 2
> `Learn -> Forward -> Flood when unknown`
