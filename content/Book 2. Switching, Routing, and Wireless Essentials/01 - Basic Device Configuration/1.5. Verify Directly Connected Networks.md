# 1.5 Verify Directly Connected Networks
## 1.5.1 Main verification commands
После настройки сети наступает вторая половина работы: **verification**.
**verification** — проверка, что конфигурация реально работает, а не просто введена в CLI. #networkterm
Главные команды:

|Command|Что делает|
|---|---|
|`show ip interface brief`|Быстро показывает IPv4 interfaces и их status|
|`show ipv6 interface brief`|Быстро показывает IPv6 interfaces и их status|
|`show running-config interface <id>`|Показывает config конкретного interface|
|`show ip route`|Показывает IPv4 routing table|
|`show ipv6 route`|Показывает IPv6 routing table|
|`ping`|Проверяет reachability до узла|
## 1.5.2 Verify interface status
Команды:
```bash
R1# show ip interface brief
R1# show ipv6 interface brief
```
`show ip interface brief` #ciscoIOScommand
Даёт краткий обзор IPv4 addresses и состояний interfaces.
`show ipv6 interface brief` #ciscoIOScommand
Даёт краткий обзор IPv6 addresses и состояний interfaces.
Как читать статус:

|Status|Что значит|
|---|---|
|`up/up`|Интерфейс работает нормально|
|`down/down`|Нет физического линка|
|`administratively down`|Интерфейс выключен командой `shutdown`|
> [!warning] Частый exam moment
> Видишь `administratively down` -> вспоминай `no shutdown`.
## 1.5.3 IPv6 link-local and multicast
На IPv6 interface ты обычно увидишь не один address, а несколько.
**link-local address** — локальный адрес интерфейса, который работает только в пределах канала. #networkterm  
Он обычно начинается с `FE80`.
**multicast address** — адрес группы узлов, а не одного хоста. #networkterm  
IPv6 multicast часто начинается с `FF`.
Команда:
```bash
R1# show ipv6 interface gigabitethernet 0/0/0
```
`show ipv6 interface gigabitethernet 0/0/0` #ciscoIOScommand
Показывает детальную IPv6 информацию по интерфейсу.
Что нужно увидеть:
- `FE80::...` = link-local;
- `2001:DB8:...` = global unicast;
- `FF02::...` = multicast groups.
> [!tip] Для памяти
> `FE80 = link-local`  
> `FF = multicast`
## 1.5.4 Verify interface configuration
Команда:
```bash
show running-config interface gigabitethernet 0/0/0
```
`show running-config interface gigabitethernet 0/0/0` #ciscoIOScommand
Показывает, какие команды применены именно к этому interface.
Это удобно, когда ты хочешь быстро проверить:
- address;
- description;
- shutdown/no shutdown;
- IPv4/IPv6 settings.
Дополнительно:
- `show interfaces` #ciscoIOScommand — полная информация и counters;
- `show ip interface` #ciscoIOScommand — IPv4 details;
- `show ipv6 interface` #ciscoIOScommand — IPv6 details.
## 1.5.5 Verify routes
Команды:
```bash
show ip route
show ipv6 route
```
`show ip route` #ciscoIOScommand
Показывает IPv4 routing table.
`show ipv6 route` #ciscoIOScommand
Показывает IPv6 routing table.
Главные коды:

|Code|Meaning|
|---|---|
|`C`|Connected|
|`L`|Local|
**Connected route** — сеть, которая напрямую подключена к interface router. #networkterm  
**Local route** — собственный адрес interface самого router. #networkterm

|Protocol|Local route length|
|---|---|
|`IPv4`|`/32`|
|`IPv6`|`/128`|
## 1.5.6 Test connectivity with `ping`
`ping` #ciscoIOScommand
Отправляет ICMP echo packets и проверяет, отвечает ли удалённый узел.
Пример:
```bash
R1# ping 2001:db8:acad:1::10
```
Если видишь:
```bash
!!!!!
```
значит echo replies пришли успешно.
**ICMP (Internet Control Message Protocol)** — протокол служебных сообщений сети, используемый в том числе для `ping`. #abbreviation
## 1.5.7 Filter `show` command output
Иногда вывод слишком длинный. Тогда помогают filters.
Синтаксис:
```bash
show <command> | <filter> <expression>
```
Filters:

|Filter|Что делает|
|---|---|
|`section`|Показывает секцию|
|`include`|Оставляет только нужные строки|
|`exclude`|Убирает ненужные строки|
|`begin`|Начинает вывод с нужного места|
Примеры:
```bash
R1# show running-config | section line vty
R1# show ip interface brief | include up
R1# show ip interface brief | exclude up
R1# show running-config | begin line
```
`section` #ciscoIOScommand  
`include` #ciscoIOScommand  
`exclude` #ciscoIOScommand  
`begin` #ciscoIOScommand
Это не отдельные "команды настройки", а фильтры CLI вывода.
> [!note] Полезно знать
> `terminal length 0` #ciscoIOScommand  
> отключает `--More--` и показывает длинный вывод без пауз.
## 1.5.8 Command history
CLI умеет хранить историю последних команд.
Команды:
```bash
R1# show history
R1# terminal history size 200
```
`show history` #ciscoIOScommand
Показывает последние введённые команды.
`terminal history size 200` #ciscoIOScommand
Увеличивает размер буфера истории.
Полезные клавиши:
- `Ctrl+P` / Up Arrow — previous command;
- `Ctrl+N` / Down Arrow — next command.
## 1.5.9 Packet Tracer summary
Если ты понял тему, то уже должен уметь не просто “смотреть команды”, а **читать их смысл**:
- посмотреть interface status;
- понять, есть ли connected routes;
- проверить `ping`;
- быстро отфильтровать нужный кусок вывода.
> [!success] Формула темы
> `Verification = interfaces + routes + ping + filters`
