# 1.1 Configure a Switch with Initial Settings
## 1.1.1 Switch boot sequence
Когда ты включаешь Cisco switch, он не сразу попадает в IOS.  
Он проходит несколько стадий загрузки.
Последовательность:
1. `POST`
2. `boot loader`
3. `CPU initialization`
4. `flash file system initialization`
5. `IOS load`
### Термины
**POST (Power-On Self-Test)** — проверка железа при включении. #abbreviation  
**boot loader** — маленькая программа в `ROM`, которая запускает дальнейшую загрузку. #networkterm  
**flash** — энергонезависимая память, где обычно лежит IOS image. #networkterm  
**IOS (Internetwork Operating System)** — ОС Cisco устройств. #abbreviation

|Stage|Что это значит простыми словами|
|---|---|
|`POST`|Устройство проверяет, живо ли железо|
|`boot loader`|Ищет и запускает IOS|
|`CPU init`|Подготавливает процессор и память|
|`flash init`|Подготавливает файловую систему flash|
|`IOS load`|Загружает ОС|
> [!tip] Запомни как цепочку
> `POST -> boot loader -> flash -> IOS`
## 1.1.2 The `boot system` command
Switch использует переменную `BOOT`, чтобы понять, какой IOS image нужно загрузить.
Новая команда:
```bash
S1(config)# boot system flash:/c2960-lanbasek9-mz.150-2.SE/c2960-lanbasek9-mz.150-2.SE.bin
```
`boot system` #ciscoIOScommand
Что делает команда:
- явно указывает путь к IOS image;
- помогает контролировать, какой образ будет загружен при следующем старте.
Проверка:
```bash
S1# show boot
```
`show boot` #ciscoIOScommand
Что делает команда:
- показывает boot-параметры;
- показывает, откуда switch собирается загружать IOS.
> [!warning] Важно
> Если `BOOT` path неправильный, switch может не загрузить нужный IOS.
## 1.1.3 Switch LED indicators
LED indicators нужны не для красоты, а для **быстрой диагностики без CLI**.
**LED** = light-emitting diode, световой индикатор. #abbreviation

|LED|Что означает|
|---|---|
|`SYST`|Общее состояние устройства|
|`RPS`|Состояние резервного питания|
|`STAT`|Состояние порта|
|`DUPLX`|Режим duplex|
|`SPEED`|Скорость порта|
|`PoE`|Питание по Ethernet|
|`Mode`|Переключает режим отображения LED|
**PoE (Power over Ethernet)** — подача питания по Ethernet-кабелю. #abbreviation  
**RPS (Redundant Power System)** — резервная система питания. #abbreviation
> [!note] Что реально спрашивают
> `DUPLX` LED связан с `half/full-duplex`,  
> `SPEED` LED связан со скоростью интерфейса.
## 1.1.4 Recover from a system crash
Если IOS не загружается, устройство можно поднять через `boot loader`.
Вход в boot loader:
1. Подключиться по `console`.
2. Перезапустить switch.
3. Удерживать кнопку `Mode`.
4. Дождаться приглашения `switch:`.
Команды:
```bash
switch: set
switch: flash_init
switch: dir flash:
switch: boot
```
`set` #ciscoIOScommand
Показывает переменные окружения, включая `BOOT`.
`flash_init` #ciscoIOScommand
Инициализирует flash file system.
`dir flash:` #ciscoIOScommand
Показывает содержимое flash.
`boot` #ciscoIOScommand
Пытается загрузить IOS вручную.
> [!tip] Как понять тему
> `boot loader` нужен как аварийный режим: посмотреть flash, найти image и попробовать загрузиться.
## 1.1.5 Management access with SVI
Чтобы управлять switch по сети, нужен **SVI**.
**SVI (Switch Virtual Interface)** — виртуальный интерфейс коммутатора для управления. #abbreviation
Это важно понять правильно:
- `SVI` не является физическим портом;
- через `SVI` switch получает management IP;
- это нужно для remote access.
Если управление идёт из другой сети, нужен ещё и **default gateway**.
**default gateway** — адрес маршрутизатора, через который устройство выходит в другие сети. #networkterm
> [!important] Главное различие
> `SVI` даёт management access,  
> но не превращает Layer 2 switch в router.
## 1.1.6 SVI configuration example
Обычно для управления делают отдельный **management VLAN**.
**VLAN (Virtual Local Area Network)** — логическое разделение сети внутри коммутатора. #abbreviation  
**management VLAN** — VLAN, используемый специально для управления устройством. #networkterm
Команды:
```bash
S1# configure terminal
S1(config)# interface vlan 99
S1(config-if)# ip address 172.17.99.11 255.255.255.0
S1(config-if)# ipv6 address 2001:db8:acad:99::11/64
S1(config-if)# no shutdown
S1(config-if)# end
S1# copy running-config startup-config
S1(config)# ip default-gateway 172.17.99.1
```
`configure terminal` #ciscoIOScommand
Переводит устройство в global configuration mode.
`interface vlan 99` #ciscoIOScommand
Открывает настройку виртуального интерфейса VLAN 99.
`ip address 172.17.99.11 255.255.255.0` #ciscoIOScommand
Назначает IPv4 address и subnet mask.
`ipv6 address 2001:db8:acad:99::11/64` #ciscoIOScommand
Назначает IPv6 address.
`no shutdown` #ciscoIOScommand
Включает интерфейс.
`end` #ciscoIOScommand
Возвращает из config mode в privileged EXEC mode.
`copy running-config startup-config` #ciscoIOScommand
Сохраняет текущую конфигурацию в startup-config.
`ip default-gateway 172.17.99.1` #ciscoIOScommand
Указывает, через какой gateway switch будет выходить в другие сети.
Проверка:
```bash
S1# show ip interface brief
S1# show ipv6 interface brief
```
`show ip interface brief` #ciscoIOScommand
Кратко показывает IPv4 interfaces и их статус.
`show ipv6 interface brief` #ciscoIOScommand
Кратко показывает IPv6 interfaces и их статус.
> [!warning] Частая ошибка
> `SVI` не станет `up/up`, если VLAN не активен и в нём нет работающего порта.
## 1.1.7 Lab connection
В lab по этой теме ты обычно:
- смотришь default config;
- настраиваешь management access;
- проверяешь connectivity;
- смотришь MAC address table.
**MAC address table** — таблица, в которой switch хранит соответствие MAC-адресов и портов. #networkterm
