# 1.4 Basic Router Configuration
## 1.4.1 Basic router setup
Настройка router очень похожа на initial setup switch.  
Смысл тот же: дать устройству имя, защитить доступ и сохранить конфигурацию.
Команды:
```bash
Router# configure terminal
Router(config)# hostname R1
R1(config)# enable secret class
R1(config)# line console 0
R1(config-line)# password cisco
R1(config-line)# login
R1(config-line)# exit
R1(config)# line vty 0 4
R1(config-line)# password cisco
R1(config-line)# login
R1(config-line)# exit
R1(config)# service password-encryption
R1(config)# banner motd $ Authorized Access Only! $
R1# copy running-config startup-config
```
`hostname R1` #ciscoIOScommand
Задаёт имя устройства. После этого prompt меняется, и ты сразу видишь, на каком устройстве работаешь.
`enable secret class` #ciscoIOScommand
Задаёт пароль для privileged EXEC mode. `secret` безопаснее, чем старый `password`.
`line console 0` #ciscoIOScommand
Переходит к настройке console line.
`password cisco` #ciscoIOScommand
Задаёт пароль на выбранной line.
`login` #ciscoIOScommand
Включает проверку пароля на line.
`line vty 0 4` #ciscoIOScommand
Переходит к настройке VTY lines для remote access.
`service password-encryption` #ciscoIOScommand
Шифрует пароли в конфигурации, которые иначе были бы видны как plain text.
`banner motd $ Authorized Access Only! $` #ciscoIOScommand
Добавляет warning banner при входе на устройство.
`copy running-config startup-config` #ciscoIOScommand
Сохраняет текущую конфигурацию.
> [!tip] Если прочитал и понял
> Initial router setup = дать имя, закрыть доступ паролями, включить шифрование и сохранить конфиг.

## 1.4.3 Dual-stack topology
**dual stack** — работа устройства одновременно с `IPv4` и `IPv6`. #networkterm
Это важная тема, потому что современная сеть часто использует оба протокола одновременно.
Если ты видишь topology с `IPv4` и `IPv6`, не нужно думать, что это две разные сети.  
Это одна и та же сеть, но с двумя адресными пространствами.
## 1.4.4 Configure router interfaces
Чтобы interface реально заработал, мало просто зайти в него.  
Нужно:
1. назначить address;
2. включить interface;
3. убедиться, что есть physical link.
Команды:
```bash
R1(config)# interface gigabitethernet 0/0/0
R1(config-if)# ip address 192.168.10.1 255.255.255.0
R1(config-if)# ipv6 address 2001:db8:acad:1::1/64
R1(config-if)# description Link to LAN 1
R1(config-if)# no shutdown
```
`interface gigabitethernet 0/0/0` #ciscoIOScommand
Открывает настройку конкретного physical interface.
`ip address 192.168.10.1 255.255.255.0` #ciscoIOScommand
Назначает IPv4 address и mask.
`ipv6 address 2001:db8:acad:1::1/64` #ciscoIOScommand
Назначает IPv6 address и prefix.
`description Link to LAN 1` #ciscoIOScommand
Добавляет подпись интерфейсу. Это полезно для чтения конфигурации и troubleshooting.
`no shutdown` #ciscoIOScommand
Включает interface.
**interface** — логическое представление порта устройства в IOS. #networkterm  
**description** — текстовая подпись, помогающая понять назначение интерфейса. #networkterm
> [!important] Очень важное правило
> Если interface не включён командой `no shutdown`, он не будет работать даже при правильной address configuration.
## 1.4.5 Interface status messages
После `no shutdown` IOS часто показывает системные сообщения:
```bash
%LINK-3-UPDOWN: Interface GigabitEthernet0/0/0, changed state to up
%LINEPROTO-5-UPDOWN: Line protocol on Interface GigabitEthernet0/0/0, changed state to up
```
Как это понять:
- `LINK ... up` = физический линк поднялся;
- `LINEPROTO ... up` = канальный уровень тоже заработал.
Это очень полезные сообщения, потому что они сразу показывают, сработала ли команда.
## 1.4.6 IPv4 loopback interfaces
**loopback interface** — виртуальный интерфейс, не связанный с физическим портом. #networkterm
Почему он полезен:
- не требует кабеля;
- обычно стабилен;
- часто используется как `router ID` или management/test address.
**router ID** — уникальный идентификатор router в некоторых routing protocols. #networkterm
> [!note] Как это понять по-человечески
> `loopback` — это “виртуальный адрес самого router”, который не зависит от кабеля и соседнего устройства.
