## 3.6.3 What did I learn in this module?

Если собрать модуль в одну картину, то должно получиться так:

- `VLAN` делит одну switched network на несколько logical broadcast domains;
- `Data VLAN`, `Voice VLAN`, `Management VLAN`, `Native VLAN` имеют разное назначение;
- `access port` обычно несёт одну data VLAN;
- `trunk port` переносит traffic нескольких VLANs;
- `802.1Q` добавляет VLAN tag;
- native VLAN используется для untagged traffic;
- ports назначаются в VLAN на switch, а не на end device;
- verification делается через `show vlan`, `show interfaces ... switchport`, `show dtp interface`.

> [!abstract] Самое важное для запоминания
> - `VLAN = logical segmentation`
> - `Trunk = multiple VLANs`
> - `802.1Q = VLAN tagging`
> - `VLAN 1 = default`
> - `switchport mode access` = access port
> - `switchport mode trunk` = trunk port
> - `switchport nonegotiate` = stop DTP negotiation
